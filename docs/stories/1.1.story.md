# Story 1.1 — Correções de UX e Lógica do Chatbot Projeta SC

**Status:** InReview  
**Epic:** 1 — Qualidade e UX do Chatbot  
**Story ID:** 1.1  
**Prioridade:** Alta  
**Estimativa:** M (3–5h de edição no n8n)

---

## Descrição

Após testes com usuários reais, foram identificados bugs de navegação, problemas de formatação, falhas no entendimento do LLM e ausência de enumeração nas listas. Esta story corrige 9 itens agrupados por tipo de mudança, todos implementados via **edição no n8n UI e exportação do JSON** (Option B).

---

## Workflow de Desenvolvimento (n8n UI — Obrigatório)

```
1. Importar docs/flow/projeta-sc.json no n8n (Settings → Import)
2. Aplicar cada change set abaixo em sequência
3. Testar no próprio n8n (test webhook ou conversa real)
4. Exportar o fluxo (Download JSON)
5. Substituir docs/flow/projeta-sc.json pelo arquivo exportado
6. Commitar
```

> ⚠️ NÃO editar o JSON diretamente. A complexidade dos nós de roteamento torna a edição visual obrigatória para evitar JSON malformado.

## Acceptance Criteria

### AC-1: "4" após visualizar contrato navega para Nova Busca
- [ ] Ao exibir 1 contrato (nó `FormadaContratos`), digitar "4" limpa o contexto e exibe a mensagem de reinício
- [ ] Ao exibir lista de contratos e clicar em um detalhe (`FormataDetalheContrato`), digitar "4" também reinicia
- [ ] Os atalhos 1/2/3/4 do menu principal NUNCA são interceptados pela seleção de contrato

### AC-2: Valor monetário formatado corretamente
- [ ] No nó `FormataProjeto`, o `VALORPLANEJADO` exibe no formato `R$ 1.000.234,00` (ponto como separador de milhar, vírgula como decimal)
- [ ] Valores zero exibem `R$ 0,00`

### AC-3: Busca fuzzy preserva nomes compostos
- [ ] Buscar "Ponte Ercilio Luz" retorna projetos da "Ponte Hercílio Luz"
- [ ] O system prompt do `InterpretaConsulta` orienta o LLM a preservar o nome exato sem corrigir ortografia

### AC-4: Seleção de município por número resolve loop
- [ ] Quando o fluxo exibe lista de municípios, as opções são numeradas (1️⃣, 2️⃣, etc.)
- [ ] O array de municípios exibidos é salvo em `contexto.municipios_opcoes`
- [ ] Em `ProcessaMensagem`, quando `contexto.municipios_opcoes` existe e o usuário digita um número válido, o município é selecionado diretamente SEM passar pelo LLM
- [ ] Digitar o nome do município ainda funciona como fallback

### AC-5: Listas de órgãos também são enumeradas
- [ ] Quando o fluxo exibe lista de órgãos como sugestão de refinamento, as opções também são numeradas
- [ ] O array de órgãos exibidos é salvo em `contexto.orgaos_opcoes`
- [ ] Seleção numérica funciona igual ao AC-4

### AC-6: "Saber mais" não aparece como texto quebrado
- [ ] Nenhum prompt enviado ao Groq/LLM menciona "Saber mais" como opção de texto
- [ ] O system prompt do `PreparaPromptRefinamento` proíbe explicitamente mencionar "Saber mais" ou links

### AC-7: "Saber mais" exibe ficha completa do projeto
- [ ] Quando o usuário digita "saber mais" (texto livre), o fluxo exibe a ficha completa do `projeto_atual`
- [ ] A ficha completa inclui: código, órgão, situação, detalhe da situação, datas, valor e execução
- [ ] O menu de opções (1/2/3/4) é exibido junto

### AC-8: "Educação" e sinônimos mapeiam para órgão correto
- [ ] Digitar "educação" resulta em busca com `orgao LIKE '%EDUC%'`
- [ ] Digitar "saúde" resulta em busca com `orgao LIKE '%SAUDE%'`
- [ ] Digitar "segurança" resulta em busca com `orgao LIKE '%SEGUR%'`
- [ ] Mapeamento implementado no nó `MontaBusca` (mapa `SINONIMOS_ORGAO`)

### AC-9: Contexto de órgão mantido nas buscas refinadas
- [ ] Ao responder "Polícia Científica" em resposta a uma pergunta sobre órgão, o classificador identifica como `orgao` (não `nome`)
- [ ] O system prompt do `GroqClassificador` incluye exemplos explícitos: `"Polícia Científica" → {tipo:"busca", campos:{orgao:"Polícia Científica"}}`

## Dev Notes — Guia de Implementação

### Fix AC-1: Reordenar steps em `ProcessaMensagem`

No código JavaScript do nó `ProcessaMensagem`, mover o bloco **"step 5 — atalhos numéricos"** para ANTES do bloco **"step 4B — seleção de contrato"**:

```
// ORDEM ATUAL (bugada):
// step 4:  seleção de projeto da lista
// step 4B: seleção de contrato  ← intercepta "4" antes do menu
// step 5:  atalhos 1/2/3/4 do menu principal

// ORDEM CORRETA:
// step 4:  seleção de projeto da lista
// step 5:  atalhos 1/2/3/4 do menu principal  ← prioridade máxima
// step 4B: seleção de contrato  ← só ativa quando não há projeto_atual
```

Condição: mover o bloco `if (eNumero && contexto.projeto_atual)` para antes do `if (eNumero && contexto.contratos && ...)`.

### Fix AC-2: Formatar valor em `FormataProjeto`

No nó `FormataProjeto`, localizar a linha:
```javascript
"💰 *Valor:* R$ " + (p.VALORPLANEJADO || "0") + "\n"
```
Substituir por:
```javascript
"💰 *Valor:* R$ " + parseFloat(String(p.VALORPLANEJADO || "0").replace(",", ".")).toLocaleString('pt-BR', { minimumFractionDigits: 2, maximumFractionDigits: 2 }) + "\n"
```

### Fix AC-4 e AC-5: Municípios e órgãos enumerados

No nó `ProcessaResultados`, função `perguntaRefinamento`, na seção `!temMunicipio`:

**Antes:**
```javascript
topMunicipios.forEach(([mun, qtd]) => {
  msg += "📍 " + mun + " — " + qtd + " projeto" + (qtd > 1 ? "s" : "") + "\n";
});
```

**Depois:**
```javascript
const emojisNum = ["1️⃣","2️⃣","3️⃣","4️⃣","5️⃣","6️⃣","7️⃣","8️⃣","9️⃣"];
staticData.contextos[remetente].municipios_opcoes = topMunicipios.map(([m]) => m);
topMunicipios.forEach(([mun, qtd], i) => {
  msg += (emojisNum[i] || (i+1) + ".") + " " + mun + " — " + qtd + " projeto" + (qtd > 1 ? "s" : "") + "\n";
});
```

Repetir o mesmo padrão para órgãos (`topOrgaos`) salvando em `contexto.orgaos_opcoes`.

No nó `ProcessaMensagem`, adicionar após o step de seleção de projeto (step 4), novos steps:

```javascript
// 4C. SELEÇÃO DE MUNICÍPIO POR NÚMERO
if (eNumero && contexto.municipios_opcoes && contexto.municipios_opcoes[parseInt(textoLimpo) - 1]) {
  const municipio = contexto.municipios_opcoes[parseInt(textoLimpo) - 1];
  contexto.campos_acumulados = { ...(contexto.campos_acumulados || {}), municipio };
  contexto.municipios_opcoes = null;
  contexto.ultimaInteracao = agora;
  staticData.contextos[remetente] = contexto;
  return [{ tipo: 'busca', busca_limpa: 'refinar', campos_acumulados: contexto.campos_acumulados, campos_faltantes: [], remetente, mensagem_retorno: '' }];
}

// 4D. SELEÇÃO DE ÓRGÃO POR NÚMERO
if (eNumero && contexto.orgaos_opcoes && contexto.orgaos_opcoes[parseInt(textoLimpo) - 1]) {
  const orgao = contexto.orgaos_opcoes[parseInt(textoLimpo) - 1];
  contexto.campos_acumulados = { ...(contexto.campos_acumulados || {}), orgao };
  contexto.orgaos_opcoes = null;
  contexto.ultimaInteracao = agora;
  staticData.contextos[remetente] = contexto;
  return [{ tipo: 'busca', busca_limpa: 'refinar', campos_acumulados: contexto.campos_acumulados, campos_faltantes: [], remetente, mensagem_retorno: '' }];
}
```

### Fix AC-7: Handler de "saber mais" em `ProcessaMensagem`

Adicionar antes do bloco `// 7. TUDO MAIS -> CLASSIFICAR COM IA`:

```javascript
// 6D. SABER MAIS — exibe ficha completa do projeto atual
const pedidoSaberMais = /^(saber mais|mais info|mais detalhes|detalhe|detalhar|ver mais)$/i.test(textoNormalizado);
if (pedidoSaberMais && contexto.projeto_atual) {
  const p = contexto.projeto_atual;
  const val = parseFloat(String(p.VALORPLANEJADO || "0").replace(",", ".")).toLocaleString('pt-BR', { minimumFractionDigits: 2, maximumFractionDigits: 2 });
  const fichaCompleta =
    "📋 *" + p.NOME + "*\n\n" +
    "📌 *Código:* " + (p.CODIGO_PROCESSO || "-") + "\n" +
    "🏛 *Órgão:* " + (p.ORGAO || "-") + "\n\n" +
    "📊 *Situação:* " + (p.SITUACAO || "-") + "\n" +
    (p.SITUACAO_ATUAL ? "✅ *Detalhe:* " + p.SITUACAO_ATUAL + "\n\n" : "\n") +
    "📅 *Início:* " + (p.DATA_INICIO || "-") + "\n" +
    "📅 *Término:* " + (p.DATA_FIM || "-") + "\n\n" +
    "💰 *Valor:* R$ " + val + "\n" +
    "📈 *Execução:* " + (p.PERCENTUAL_EXEC || "0") + "%\n\n" +
    "*Deseja mais informações?*\n" +
    "1️⃣ Ver Entregas\n2️⃣ Ver Valores\n3️⃣ Ver Contratos\n4️⃣ Nova Busca";
  contexto.ultimaInteracao = agora;
  staticData.contextos[remetente] = contexto;
  return [{ tipo: 'reinicio', remetente, mensagem_retorno: fichaCompleta }];
}
```
