# Story 1.5 — Paginação de Municípios ("Outros municípios")

**Status:** InReview  
**Epic:** 1 — Qualidade e UX do Chatbot  
**Story ID:** 1.5  
**Prioridade:** Alta  
**Estimativa:** M (3–5h de edição no n8n)

---

## Descrição

Quando o bot lista os municípios de uma busca e há mais de 5, exibe os 5 principais e agrupa o restante como `"📍 Outros municípios — N projetos"`. Ao responder `"Outros municípios"`, o bot trata essa string como um nome de município real, aplica o filtro `Município: Outros municípios` e retorna zero resultados — deixando os projetos restantes completamente inacessíveis.

Versões anteriores do bot tinham um `"saiba mais"` que exibia a lista expandida. Esta story restaura esse comportamento: quando o usuário seleciona `"Outros municípios"` (ou `"outros"`, `"mais"`, `"saiba mais"`), o bot exibe a lista completa de municípios com projetos para aquela busca.

**Valor:** Elimina um caminho morto no fluxo — hoje o usuário que clica em "Outros municípios" sempre recebe um erro, tornando 9 (ou mais) projetos inacessíveis por aquela rota.

---

## Workflow de Desenvolvimento

```
1. Importar docs/flow/projeta-sc.json no n8n (Settings → Import)
2. Identificar o nó que processa a resposta de município (provavelmente RoteiaSubmenu ou ProcessaMensagem)
3. Adicionar detecção de "outros municípios" antes de aplicar o filtro
4. Implementar lógica de exibição da lista expandida de municípios
5. Testar via webhook com os cenários de teste
6. Exportar (Download JSON) e substituir docs/flow/projeta-sc.json
7. Commitar
```

> ⚠️ Os nós de roteamento (Switch/IF) NÃO devem ser editados diretamente no JSON. Os nós Code podem ser editados no JSON com segurança.

---

## Arquitetura da Solução

```
ProcessaMensagem
  → detecta resposta = "outros municípios" / "outros" / "mais" / "saiba mais"
    → ExibeMunicipiosCompletos [NOVO — Code]
      → lê lista completa de municípios do staticData.contextos[remetente]
      → formata e exibe todos os municípios não listados anteriormente
      → aguarda nova seleção de município
  → (outros casos) fluxo atual continua normalmente
```

## Acceptance Criteria

### AC-1: Salvar lista completa de municípios no contexto

- [ ] No nó que gera a mensagem de seleção de município (ex: `ProcessaResultados` ou `FormataRefinamento`), salvar no `staticData.contextos[remetente]` a lista completa de municípios com contagem:
  ```json
  {
    "municipios_busca": [
      {"municipio": "Florianópolis", "count": 3},
      {"municipio": "Joinville", "count": 2},
      ...
    ]
  }
  ```
- [ ] A lista inclui TODOS os municípios da busca atual, não apenas os 5 principais

### AC-2: Detecção de "Outros municípios" no processamento de resposta

- [ ] Em `ProcessaMensagem` (ou nó equivalente que lê a resposta do usuário), adicionar verificação:
  - Se a mensagem do usuário corresponder a (case-insensitive, sem acentos): `"outros municípios"`, `"outros municipios"`, `"outros"`, `"mais"`, `"saiba mais"` — E — o contexto estiver no estado de seleção de município
  - Então: redirecionar para o novo nó `ExibeMunicipiosCompletos`
- [ ] A detecção não afeta outros estados do fluxo (ex: resposta "outros" em outro contexto não é interceptada)

### AC-3: Nó `ExibeMunicipiosCompletos` (Code)

- [ ] Lê `municipios_busca` do `staticData.contextos[remetente]`
- [ ] Formata mensagem com TODOS os municípios, ordenados por contagem decrescente:
  ```
  📍 *Todos os municípios com projetos "Via Mar":*
  
  1️⃣ Florianópolis — 3 projetos
  2️⃣ Joinville — 2 projetos
  3️⃣ São Martinho — 2 projetos
  4️⃣ Imaruí — 2 projetos
  5️⃣ Tubarão — 2 projetos
  6️⃣ Rio do Sul — 2 projetos
  7️⃣ Chapecó — 2 projetos
  8️⃣ Blumenau — 2 projetos
  9️⃣ Criciúma — 2 projetos
  🔟 Lages — 1 projeto
  
  De qual município você quer ver os projetos?
  ```
- [ ] Numeração sequencial até 14 itens (limite de WhatsApp); se mais de 14 municípios, agrupar os excedentes novamente
- [ ] Após exibir, aguarda nova seleção de município pelo mesmo fluxo atual

### AC-4: Seleção de município da lista expandida funciona normalmente

- [ ] Após o usuário selecionar um município da lista expandida, o fluxo continua normalmente (filtra e exibe projetos daquele município)
- [ ] A seleção aceita tanto o número (ex: `"6"`) quanto o nome do município

## Cenários de Teste

| Ação do usuário | Estado esperado | Resultado esperado |
|----------------|-----------------|-------------------|
| Busca `"via mar"` → 5 municípios + "Outros municípios — 9" | — | Lista com 5 + "Outros" exibida |
| Responde `"Outros municípios"` | contexto = seleção município | Lista expandida com todos os 14 municípios |
| Responde `"outros"` | contexto = seleção município | Lista expandida exibida |
| Responde `"saiba mais"` | contexto = seleção município | Lista expandida exibida |
| Seleciona `"Rio do Sul"` da lista expandida | — | Projetos de Rio do Sul exibidos |
| Responde `"outros"` fora do contexto de município | outro estado | Fluxo normal (não interceptado) |

## Escopo

**IN:**
- Salvar lista completa de municípios no contexto após cada busca que retorna múltiplos municípios
- Detecção de "outros municípios"/"outros"/"mais"/"saiba mais" no estado de seleção de município
- Novo nó `ExibeMunicipiosCompletos` (Code)

**OUT:**
- Paginação de municípios por página (ex: "próximos 5") — fora do escopo desta story
- Municípios com zero projetos — não listados
- Mudanças nos nós Switch/IF de roteamento principal

## Riscos

- **Estado de contexto perdido:** Se o usuário sair e voltar, `municipios_busca` pode não estar mais no `staticData`. Mitigação: a lista é gerada a cada nova busca, portanto o usuário pode simplesmente repetir a consulta.
- **Mais de 14 municípios:** WhatsApp tem limite prático de mensagem longa. Mitigação: AC-3 limita a 14 e agrupa excedentes (raro — maioria das buscas retorna < 14 municípios).
- **Colisão de detecção:** A palavra `"outros"` pode aparecer em outros contextos. A verificação de AC-2 exige que o contexto seja especificamente de seleção de município.
